# Story Studio

A Windows desktop app for generating children's story videos — paste a story, get a fully narrated and illustrated MP4 with AI-generated watercolor artwork.

Built with Electron + React. Designed for making Dutch and Mandarin YouTube kids content.

---

## What it does

1. **Split** — paste a story text, the app splits it into 5–15 scenes
2. **Illustrate** — each scene gets a watercolor children's book illustration generated by the Gemini API (Nano Banana), with narration text embedded in the image
3. **Narrate** — text-to-speech via XTTS v2 (local voice clone) or ElevenLabs (cloud voice clone)
4. **Export** — FFmpeg assembles all scenes into a 1920×1080 MP4 with xfade transitions

---

## Stack

| Layer | Tech |
|---|---|
| Desktop shell | Electron 28 |
| UI | React 18 + Vite 5 + Tailwind CSS |
| Illustration | Gemini `gemini-3.1-flash-image-preview` (Google AI Studio) |
| TTS (local) | XTTS v2 via Python Flask server |
| TTS (cloud) | ElevenLabs REST API with in-app voice cloning |
| Video | FFmpeg (bundled) |
| State | Zustand + React Query |

## Requirements

- **Windows 10/11**
- **Node.js** v18+

- **FFmpeg** — download `ffmpeg-essentials` from [gyan.dev](https://www.gyan.dev/ffmpeg/builds/)

## Setup

### 1. Clone and install

```bash
git clone https://github.com/Ahaxin/story-studio.git
cd story-studio
npm install
```

### 2. Place binaries in `resources/`

```
resources/
  ffmpeg.exe          ← from gyan.dev ffmpeg-essentials build
  ffprobe.exe         ← bundled with ffmpeg download
  piper.exe           ← from github.com/rhasspy/piper (optional)
  piper/models/       ← .onnx + .onnx.json voice model files (optional)
  xtts_server.py      ← already in repo
```

**Piper voice models** (optional, Dutch/Mandarin):
- `nl_NL-mls-medium.onnx` + `.onnx.json` — from [huggingface.co/rhasspy/piper-voices](https://huggingface.co/rhasspy/piper-voices)
- `zh_CN-huayan-medium.onnx` + `.onnx.json`

### 3. Set up API keys

Copy `.env.example` to `.env` and fill in your keys:

```bash
cp .env.example .env
```

```env
NANO_BANANA_API_KEY=your_google_ai_studio_key
GOOGLE_TTS_API_KEY=your_google_cloud_tts_key   # optional
```

API keys for ElevenLabs and Nano Banana can also be entered in the app's Settings page — they are saved to the local Electron store (never committed to git).

### 4. Set up XTTS (optional, local voice clone)

```bash
cd resources
python -m venv xtts_venv
xtts_venv\Scripts\activate
pip install -r requirements.txt
```

XTTS starts automatically when the app launches. It loads on first use (~30 seconds).

### 5. Run

```bash
npm run dev
```

## Project structure

```
electron/
  main.js       — IPC handlers, XTTS server spawn, file I/O
  preload.js    — contextBridge: exposes window.electronAPI
src/
  App.jsx       — shell: Sidebar + page router
  components/   — SceneEditor, SceneList, Sidebar, VoiceRecorder, NewStoryModal
  pages/        — StoryEditor, ExportPage, Settings
  hooks/        — useIPC.js (React Query wrappers for all IPC calls)
  store/        — useStore.js (Zustand: navigation, scene selection)
  utils/
    ffmpeg.js       — FFmpeg filter_complex video assembler
    nanoBanana.js   — Gemini illustration API client + prompt builder
    tts.js          — Piper / Google Cloud TTS / XTTS engines
    storySplitter.js — paragraph → scene splitting
    schema.js       — createProject() + createScene() factories
resources/
  xtts_server.py    — Flask server wrapping Coqui XTTS v2
  requirements.txt  — Python deps for XTTS
```

## Notes

- All API keys are stored in Electron's local store (`electron-store`) — never in project files
- Story projects and voice samples are saved locally and excluded from git
- XTTS requires Python with specific pinned dependencies (see `requirements.txt`) — do not upgrade
- FFmpeg 8.x on Windows: use `font=Arial` (fontconfig) in drawtext, not `fontfile=C:/...` (drive-letter colon breaks the filter parser)
