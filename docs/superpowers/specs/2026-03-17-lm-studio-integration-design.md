# LM Studio Integration — Design Spec
**Date:** 2026-03-17
**Status:** Draft
**Project:** Story Studio

---

## Overview

Integrate LM Studio (local OpenAI-compatible LLM server) into Story Studio to:
1. Generate a full children's story (paragraphs + illustration prompts) from a brief user idea
2. Generate or regenerate an illustration prompt for a single scene

LM Studio runs at `http://127.0.0.1:1234` and exposes the OpenAI `/v1/chat/completions` and `/v1/models` endpoints. All generated content is in the project's target language (nl-NL or zh-CN), not English.

---

## Goals

- Users can describe a story idea in a few sentences and get a complete, split story ready to illustrate
- Illustration prompts are generated by the LLM (better quality than the rule-based `buildScenePrompt`)
- Existing "paste your own story" flow is unchanged
- LM Studio status is visible at all times so users know if the feature is available

---

## Non-Goals

- Streaming responses
- Configurable LM Studio URL or model selection (hardcoded to `http://127.0.0.1:1234`)
- Story editing / rewriting after generation (user edits scene text manually as today)
- Full story regeneration from SceneEditor (only per-scene illustration prompt)
- Scene count slider in UI — fixed at 8 story-body scenes
- Auto-polling of LM Studio status — manual refresh only (badge click / Recheck button)

---

## Architecture

### New utility: `src/utils/lmStudio.js`

CommonJS module. Three exported functions:

#### `checkLmStudioStatus()`
```
GET http://127.0.0.1:1234/v1/models
→ { online: boolean, modelId: string | null }
```
Returns `online: false` on any network error (not throw). `modelId` is the first model's ID from the response, or `null` if none loaded.

#### `generateStory({ idea, language, sceneCount = 8 })`
```
POST http://127.0.0.1:1234/v1/chat/completions
→ { scenes: [{ text: string, illustrationPrompt: string }] }
```
- Returns only **story-body scenes** — no cover. The cover scene (index 0) is always synthesised from the story title by `handleCreate()` in the existing flow, unchanged.
- `sceneCount` is the number of story-body paragraphs (default 8, so 9 total scenes including cover).
- System prompt instructs the model to write a children's story in the target language with exactly `sceneCount` scenes (paragraphs, 20–40 words each), safe for ages 4–10
- On JSON parse failure: throws a clear error message
- On scene count mismatch (LLM returned fewer/more than requested): **clamp silently** to 5–15, show a warning banner in the UI if delta > 2
- Uses `axios` (already a dependency in the project) for the HTTP call
- Timeout: 60 seconds

#### `generateIllustrationPrompt({ sceneText, language, storyTitle, illustrationStyle })`
```
POST http://127.0.0.1:1234/v1/chat/completions
→ string (illustration prompt)
```
- Focused prompt: given the scene text, story title, and illustration style context, return a single vivid illustration prompt for a watercolor children's book
- `illustrationStyle` comes from `project.style.illustrationStyle` — included in the prompt to keep visual style consistent
- `storyTitle` included in the prompt for narrative context
- Uses `axios` for the HTTP call
- Timeout: 30 seconds

### New IPC channels (in `electron/main.js`)

| Channel | Args | Returns |
|---|---|---|
| `lmstudio:status` | none | `{ online, modelId }` |
| `lmstudio:generate-story` | `{ idea, language }` | `{ scenes: [{text, illustrationPrompt}] }` |
| `lmstudio:generate-prompt` | `{ sceneText, language, storyTitle, illustrationStyle }` | `{ prompt: string }` |

### New React hooks (in `src/hooks/useIPC.js`)

- `useLmStudioStatus()` — React Query with `staleTime: 0`, refetch on mount only (no polling interval — manual refresh via badge click and Recheck button is intentional per non-goals)
- `useGenerateStory()` — mutation
- `useGeneratePrompt()` — mutation

Add a tab toggle above the textarea:

```
[ ✍️ Paste Story ]  [ ✨ Write for Me ]
```

**"Paste Story" tab:** existing textarea + Split button, unchanged.

**"Write for Me" tab:**

- Textarea: placeholder "Describe your story idea in a few sentences…"
- "Generate Story" button — disabled when LM Studio offline or idea is empty
- Loading state: spinner + "Writing your story…"
- On success: renders the same split preview as the paste flow (scene list). Note: "Illustration prompts generated by AI"
- On error: red banner with message + "Try Again" button. Collapsed `<details>` with raw response for debugging.
- Generated scenes pass through existing `handleCreate()` unchanged

### `SceneEditor.jsx` — Illustration prompt field

Add a small **"✨ AI"** icon button next to the illustration prompt textarea label.

- Disabled when LM Studio offline
- On click: spinner, calls `lmstudio:generate-prompt`, writes result via `setLocalPrompt(result)` **and** sets `promptDirty = true` — this triggers the existing 1-second auto-save timer so the prompt is persisted to `project.json` without requiring the user to type
- Does NOT auto-trigger illustration generation
- User reviews/edits the prompt, then clicks "Generate Illustration" as before

### `Settings.jsx` — LM Studio status section

New read-only section under API Keys:

```
  ● Online — mistral-7b-instruct    [Recheck]

```

## Prompt Engineering

### Story generation system prompt

```
You are a children's book author. Write a children's story in {LANGUAGE_NAME}
based on the following idea. The story must have exactly {N} scenes (paragraphs).
Each paragraph should be 20–40 words, warm, and safe for children aged 4–10.

Return ONLY valid JSON with no other text, in this exact shape:
{
  "scenes": [
    { "text": "...", "illustrationPrompt": "..." },
    ...
  ]
}

For each scene, "illustrationPrompt" should be a vivid, specific description
suitable for a watercolor children's book illustration — describe the setting,
characters, mood, and action in 1–2 sentences.
```

Language name mapping: `nl-NL` → "Dutch", `zh-CN` → "Chinese (Simplified)".
Default scene count: 8 (mid-range of allowed 5–15).

### Per-scene illustration prompt

```
You are an art director for a children's picture book titled "{storyTitle}".
The illustration style is: {illustrationStyle}.
Given the following story scene text, write a single illustration prompt
for a watercolor children's book illustration in that style. Describe the setting,
characters, mood, lighting, and action in 1–2 sentences. Be specific and vivid.
Safe for children aged 4–10. No scary elements.

Scene text: "{sceneText}"

Return only the illustration prompt, no other text.
```

## Error Handling

| Situation | User-visible message |
|---|---|
| LM Studio offline at click time | "LM Studio is not running. Start it and load a model first." |
| JSON parse failure | "Model returned invalid response. Try again." + collapsed raw response |
| Scene count out of 5–15 after generation | Clamp to 5–15, show warning banner |
| Network timeout (60s story / 30s prompt) | "LM Studio took too long to respond." |
| Model returns fewer/more scenes than requested | Clamp to 5–15 silently; show warning banner in UI if delta > 2 |

## Data Flow

```
User enters idea
  → "Generate Story" clicked
  → lmstudio:generate-story IPC
    → lmStudio.generateStory()
      → POST /v1/chat/completions
      → parse JSON response
      → clamp scene count to 5–15
    → return { scenes: [{ text, illustrationPrompt }] }
  → renderer stores result as { splitResult: paragraphs[], aiPrompts: prompts[] }
  → handleCreate() called — prepends cover scene from story title (UNCHANGED)
    → passes scenes array to story:create IPC as:
        [ coverScene, ...storyScenes ]
      where storyScenes include the LM-generated text AND the illustrationPrompt
        (passed as a field alongside text/index/narrator/transition)
  → story:create IPC handler:
      → calls createScene(index, text, narrator, transition) for each scene
        (createScene() initialises illustrationPrompt: '')
      → IMMEDIATELY AFTER createScene(), patches scene.illustrationPrompt
        with the value passed from the renderer if present
      → writes project.json
```

**Key detail — required code changes:**

1. `createScene()` always initialises `illustrationPrompt: ''`. The current `story:create` IPC handler calls `createScene()` and then immediately calls `ensureIllustrationPrompts()`, which overwrites empty prompts with `buildScenePrompt()` output. **The `story:create` handler must be modified** so that after `createScene()` runs for each scene, it patches `scene.illustrationPrompt = incomingScene.illustrationPrompt` when the incoming field is non-empty — before `ensureIllustrationPrompts()` runs. `ensureIllustrationPrompts()` already skips non-empty prompts, so LM-generated prompts will be preserved once the patch is in place.

2. `handleCreate()` in `NewStoryModal.jsx` currently constructs `storyScenes` with only `{ text, index, narrator, transition }`. **`handleCreate()` must be modified** to also include `illustrationPrompt` in each scene object passed to the IPC, regardless of which tab was used. For the paste flow, `illustrationPrompt` will be `''` (preserving existing behaviour). For the "Write for Me" flow, it will be the LM-generated prompt.
